Latest release v1.2 (1st September 2013)

Release Notes

v1.2 (1st September 2013)

  • Improvement #46
    The validation now also checks that persons in the registrations sheet are distinguishable from each other.
  • Improvement #47
    The validation now also checks that each result can be linked to a person in the registration sheet.
  • Improvement #49
    The validation now also checks that persons inside one round are distinguishable from each other.

v1.1.1 (23rd April 2013)

  • Bug fix #42
    Updating the WCA database export didn't work if no database export was downloaded previously. This has now been fixed.
  • Bug fix #43
    Rows with only DNS results where not always reported as a validation error in combined rounds. This has now been fixed.

v1.1 (16th April 2013)

  • Improvement #37
    The validation now also checks for wrongly sorted results.
  • Improvement #38
    The validation now also checks for wrongly calculated positions.
  • Improvement #12
    The validation now also checks for wrongly calculated multiple blindfolded scores.
  • Improvement #9
    The validation now also checks for rows with only DNS results.
  • Improvement #39
    Mistakes in names and countries no longer prevent the user from generating database scripts. This is useful when the name or country in the database is wrong.
  • Improvement #41
    The WCA database export can be downloaded directly from the application and the application automatically checks for updates to the database export on startup.
  • Bug fix #24
    Sheets in seconds were sometimes recognized as numbers if the first result happened to be a whole number. This has now been fixed.

v1.0 (13th January 2013)

  • Bug fix #15
    The validation now checks that all results in a sheet have the expected result format.
  • Improvement #7
    Results can now be exported to JSON for automated uploading.
  • Improvement #8
    The validation now also checks WCA ID's and names based on WCA database exports.
  • Improvement #26
    The validation now also checks for invisible 3rd digits after the decimal point.
  • Improvement #30
    The validation now also checks for duplicate event/round combinations.
  • Improvement #32
    Validation errors now refer to the cell address instead of just the row number.
  • Improvement #33
    Validation errors are now displayed in a table instead of a flat list.
  • Improvement #36
    Validation errors panel now has an option to show all errors in all sheets.
  • Improvement #34 and #35
    Better window icons.

v0.3 (2nd December 2012)

  • Bug fix #13
    The validation kept validating results when the result format was invalid, leading to misleading validation errors. This is now fixed.
  • Bug fix #17
    The Sheets contents pane sometimes didn't show the best and average results. This is now fixed.
  • Bug fix #22
    The recognition of the result format for multiple blindfolded was sometimes incorrect if the round had only DNF results. This is now fixed.
  • Bug fix #25
    Workbooks that contained cell formats with Locale information didn't always open properly. This is now fixed.
  • Improvement #6
    The Persons query now only updates persons who competed in the uploaded competition.
  • Improvement #27
    The Persons query now also returns the persons who still have a missing date of birth or gender.

v0.2 (29th October 2012)

  • Bug fix #2
    The workbook file used to be locked by the application under Windows, which prevented making changes using Excel and refreshing from the application. This is now fixed.
  • Bug fix #5
    The validation didn't check that non-empty results are formatted correctly for combined rounds. This is now fixed.
  • Improvement #3
    The validation now also checks for wrongly calculated best and average results.
  • Improvement #4
    The validation now also checks for rounding of times >10 minutes.

v0.1 (21st August 2012)

  • Initial release
